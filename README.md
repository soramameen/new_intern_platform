# インターン生・企業マッチングサービス 機能確認ガイド

このガイドでは、インターン生と企業をマッチングする Web アプリケーションの機能を、**インターン生**と**企業**の両方の立場から順を追って確認する手順を説明します。

## アプリケーションへのアクセス

本アプリケーションは既に本番環境にデプロイ済みです。以下の URL からアクセスしてください：

```
https://new-intern-platform.vercel.app/
```

## インターン生としての機能確認フロー

### 1. アカウント登録（サインアップ）

1. トップページにアクセスし、「今すぐ登録」ボタンをクリックします。
2. ユーザータイプで「インターン生」を選択します。
3. 以下の情報を入力します：
   - メールアドレス: （任意のメールアドレス。例: `test-intern@example.com`）
   - パスワード: （任意のパスワード。例: `password123`）
   - パスワード（確認）: （同じパスワードを再入力）
   - 名前: （任意の名前。例: `テスト太郎`）
   - スキル: （任意のスキル。例: `JavaScript, React, Ruby`）
   - 自己紹介: （任意の自己紹介文）
4. 「アカウント登録」ボタンをクリックします。

### 2. ダッシュボードの確認

登録成功後、自動的にダッシュボードに移動します。ここで確認できる内容：

- 自分のプロフィール情報のサマリー
- 登録されている企業の一覧
- 各種機能へのナビゲーションリンク

### 3. プロフィール詳細の編集

1. ナビゲーションメニューから「プロフィール」をクリックします。
2. 「編集する」ボタンをクリックします。
3. 以下の情報を入力/更新します：
   - 学校名
   - 専攻
   - 卒業予定年月
   - 希望勤務地
   - GitHub URL
   - ポートフォリオ URL
   - 自己紹介（詳細）
4. 「保存する」ボタンをクリックします。
5. 更新されたプロフィール情報を確認します。

### 4. 企業の閲覧

1. ダッシュボードに戻ります。
2. 企業一覧から詳細を確認したい企業の「詳細を見る」ボタンをクリックします。
3. 企業詳細ページで、以下の情報を確認します：
   - 会社名
   - 業種
   - 企業規模
   - 所在地
   - Web サイト
   - 会社概要
4. 「一覧に戻る」ボタンをクリックしてダッシュボードに戻ります。

### 5. オファーの確認と対応

※ここでは、企業からオファーが送られていることを前提とします。まだ企業側の操作を行っていない場合は、後の「企業としての機能確認フロー」を先に実施してオファーを送信してください。

1. ナビゲーションメニューから「受信したオファー」をクリックします。
2. 受信したオファーの一覧が表示されます。
3. 各オファーの詳細を確認し、「承諾する」または「辞退する」ボタンをクリックしてオファーに対応します。
4. 対応後、オファーのステータスが変更されたことを確認します。

### 6. グループチャットの利用

1. ナビゲーションメニューから「グループチャット」をクリックします。
2. これまでのメッセージを確認します。
3. 「ニックネーム」欄にニックネームを入力します（デフォルトではログイン名が設定されています）。
4. 「メッセージ」欄に送信したいメッセージを入力します。
5. 「送信」ボタンをクリックします。
6. メッセージが投稿され、チャット履歴に表示されることを確認します。

### 7. ログアウト

1. 画面右上の「ログアウト」ボタンをクリックします。
2. ログイン画面に戻ることを確認します。

## 企業としての機能確認フロー

### 1. アカウント登録（サインアップ）

1. トップページにアクセスし、「今すぐ登録」ボタンをクリックします。
2. ユーザータイプで「企業」を選択します。
3. 以下の情報を入力します：
   - メールアドレス: （任意のメールアドレス。例: `test-company@example.com`）
   - パスワード: （任意のパスワード。例: `password123`）
   - パスワード（確認）: （同じパスワードを再入力）
   - 名前: （担当者名。例: `採用担当`）
   - 会社名: （任意の会社名。例: `テスト株式会社`）
   - 会社概要: （任意の会社概要）
4. 「アカウント登録」ボタンをクリックします。

### 2. ダッシュボードの確認

登録成功後、自動的にダッシュボードに移動します。ここで確認できる内容：

- インターン生一覧
- スキルフィルター機能
- 企業プロフィール編集へのリンク
- オファー管理へのリンク

### 3. 企業プロフィールの編集

1. ナビゲーションメニューから「企業プロフィール」をクリックします。
2. 「編集する」ボタンをクリックします。
3. 以下の情報を入力/更新します：
   - 会社名
   - 業種
   - 企業規模
   - 所在地
   - Web サイト
   - 会社概要
4. 「保存する」ボタンをクリックします。
5. 更新されたプロフィール情報を確認します。

### 4. インターン生の検索とフィルタリング

1. ダッシュボードに戻ります。
2. スキルフィルターを使用して特定のスキルを持つインターン生を検索します：
   - 「スキルで絞り込み」ドロップダウンから任意のスキルを選択します。
   - フィルタリングされたインターン生一覧を確認します。
3. 「リセット」ボタンをクリックしてフィルターをクリアします。

### 5. インターン生の詳細確認とオファー送信

1. インターン生一覧から詳細を確認したいインターン生の「詳細を見る」ボタンをクリックします。
2. インターン生詳細ページで、以下の情報を確認します：
   - 名前
   - スキル
   - 学校名
   - 専攻
   - 卒業予定年月
   - 希望勤務地
   - GitHub URL（設定されている場合）
   - ポートフォリオ URL（設定されている場合）
   - 自己紹介
3. 「オファーを送る」ボタンをクリックします。
4. オファーフォームに以下の情報を入力します：
   - ポジション（例：「フロントエンドエンジニア」）
   - メッセージ（例：「あなたのスキルセットに興味を持ちました。」）
   - 詳細情報（任意。例：「週 3 日、リモートワーク可能、時給 1,500 円〜」）
5. 「送信する」ボタンをクリックします。
6. オファー送信成功のメッセージを確認します。

### 6. 送信したオファーの管理

1. ナビゲーションメニューから「送信したオファー」をクリックします。
2. 送信したオファー一覧を確認します：
   - インターン生情報
   - ポジション
   - メッセージ
   - 状態（保留中、承諾済み、拒否）
   - 送信日
3. さきほど送信したオファーが一覧に表示されていることを確認します。

### 7. ログアウト

1. 画面右上の「ログアウト」ボタンをクリックします。
2. ログイン画面に戻ることを確認します。

## 注意事項とトラブルシューティング

**アプリにアクセスできない場合**:

- インターネット接続を確認してください。
- 別のブラウザで試してみてください。

**ログインできない場合**:

- メールアドレスとパスワードが正確に入力されているか確認してください。
- 新しいアカウントを登録して試してみてください。

**表示が遅い場合**:

- 初回アクセス時はバックエンドサーバーが「スリープ」状態から復帰するため、少し時間がかかる場合があります。数分待ってから再度試してください。

# サービス説明

## 技術スタック

### フロントエンド

- Next.js 15.2
- React 19.0
- TypeScript
- Tailwind CSS
- 認証管理: JWT + localStorage

### バックエンド

- Ruby on Rails
- SQLite（開発環境）
- PostgreSQL
- JWT 認証
- RESTful API

### その他

- ChatGPT 4.5 など
- Claude sonnet 3.7 など
- Grok3
- VSCode
- Github Copilot
- Vercel ( Next.js ホスティング用 )
- render ( Ruby on rails ホスティング用 )

## 主な機能

- **ユーザー認証システム**：JWT 認証を用いた安全なログイン・ログアウト機能
- **ユーザー種別管理**：インターン生と企業で異なる権限とビュー
- **一覧取得**：インターン生は企業一覧、企業はインターン生一覧を取得可能
- **プロフィール管理**：
  - **インターン生**：スキル、学歴、自己紹介等の登録・編集
  - **企業**：会社情報、業種等の登録・編集
- **オファー機能**：
  - **企業**：企業から気になるインターン生にオファーを送ることが可能
  - **インターン生**：企業から受け取ったオファーを承認・否認が可能
- **掲示板機能**
  - **インターン生**同士の掲示板機能。ニックネームとメッセージを投稿

## アピールポイント

1. **セキュアな認証システム**

   - JWT を使用したトークンベースの認証システムを実装
   - クライアント側でのトークン管理とサーバーサイドでの検証による二重のセキュリティ
   - ユーザーのロールベースのアクセス制御（企業とインターン生で異なる権限）

2. **UX/UI の工夫**

   - ユーザーフレンドリーなダッシュボード設計
   - Tailwind CSS を活用した一貫性のあるデザイン
   - ユーザーフィードバックの視覚的表示（エラーメッセージ、ローディング状態等）

3. **掲示板(chat)**

   - インターン生同士の掲示板機能
   - 一定時間ごとの更新
   - WebSocket は今回使用していない

## 開発で苦労した点と解決策

1. **rails での認証機能の実装**

   - **課題**：**devise** を用いた認証機能の実装がうまくいかなかった．
   - **解決策**：
     - デフォルトで rails で提供されている機能を使い，
       最小限の認証でプロトタイプとして作成した．

2. **Next.js 側での認証状態の管理**

   - **課題**：Next.js での認証機能の管理がうまくいかなかった．
   - **解決策**：
     - **jwt** を用いてシンプルに実装することにした．
     - **Local storage** にトークンとユーザー情報を保存しておくことで
       どこでも呼び出せるようにした
     - **utils.ts** に Local storage とのやり取りができる関数をまとめている

3. **異なるユーザータイプの処理**

   - **課題**：インターン生と企業で異なるプロフィール情報や UI が必要だった．それぞれで認証機能を作ろうとしたが，難しかった．
   - **解決策**：
     - user モデルに intern か company かのカラムを追加することによって解決．
     - 異なるユーザータイプ用のページにアクセスした際，リダイレクトされるように．
     - 共通のページはユーザータイプで分岐するように．

4. **ホスティングのためのデプロイ**

   - **課題**：開発環境よりも厳しい文法チェックにより error 多発。

   - **解決策**：型付け不足に対する error については type.ts に Interface をまとめて保存することで対処。any を全て書き換える。

5. **Next.js のバージョンアップによる error**

   - **課題**：AI や Qiita を参考にしながら作ると、Page Router,App Router の違い，Next.js 14,15 の細かな仕様の違いに苦しんだ。

   - **解決策**：14 から 15 への以降 error はエラー出力に丁寧に書かれている場合が多く，それらを修正することで解決。
     error をそのままググっても見つかることが多かった。
